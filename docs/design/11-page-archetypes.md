# Page archetypes

**Purpose:** Map recurring **experience jobs** to compositional patterns so new routes inherit the same experiential intelligence—not the same React components.

**Audience:** Anyone authoring or generating pages; archetypes are stack-agnostic.

---

## How to use archetypes

Archetypes answer:

- What is the **primary user intent** on entry?
- What is the **pacing curve** (slow magazine vs. fast inventory)?
- Where does **operational density** belong without killing immersion?

Pick **one** primary archetype per route. Layer secondary patterns only when the user explicitly switches mode (e.g. from discovery to comparison).

---

## 1. Magazine / home orientation

**Job:** Establish taste, worldview, and breadth—**not** immediate task completion.

**Pacing:** Slow → expansive → thematic clusters → tool or map → thematic closure.

**Structure:**

1. **Immersion hero** — one strong thesis line + supporting promise (not a feature matrix).
2. **Curated highlights** — small parity grid of “faces” or entry points.
3. **Thematic band** — one topic (food, culture) with story cards; elevated density vs. hero but still editorial.
4. **Visual destinations** — tall imagery blocks for **evocative scanning**.
5. **Participatory or spatial layer** — map, explorer, interactive affordance *after* emotional priming.
6. **Collections / themes** — grouped narratives for return visits.
7. **Closing chapter** — seasonal, practical, or “what now” with **human** framing (not a coupon).

**Why this order:** Maps and tools presented too early read as **utility chrome**; placed mid-to-late, they read as **invitation**.

---

## 2. Discovery list (browse + filter)

**Job:** Frame a collection; let users **narrow** without feeling trapped in a catalog.

**Pacing:** Hero promise → **mood framing** (collections/themes) → explicit browse controls → grid.

**Structure:**

1. **List hero** — category promise + tone (“how we curate”), not item count as the headline.
2. **Collection / mood band** — large selectable tiles that teach **how to read** the catalog.
3. **Browse header** — filters and sort as **soft** controls; copy explains philosophy of filtering.
4. **Parity card grid** — items share card altitude and interaction grammar.

**Why:** Filters before mood framing feel administrative; mood framing teaches categories **before** users touch machinery.

---

## 3. Long-form detail (story + proof + plan)

**Job:** Replace brochure listing with **trust-building narrative** that still lands commercial facts.

**Pacing:** Full-viewport immersion → **orientation strip** → editorial blocks → operational bands → proof → plan → related.

**Canonical curve:** feel → **honest logistics** → **see/hear proof** → **decide** → **continue wandering**.

**Detail variants:**

- **Experience product** (tour, retreat): itinerary as **arc**, voices before or beside ratings depending on brand trust strategy.
- **Place product** (hotel, room): inventory previews after story; policies as **bounded** blocks, not legal wallpaper.

**Why:** Same archetype, different **density placement**—inventory sits after “why stay,” not before.

---

## 4. Place / destination dossier (conditional depth)

**Job:** City or region page as **living guide**—may be incomplete.

**Pacing:** Same hero grammar as detail; **branch** on content maturity.

**Structure:**

- **Full guide:** food → experiences band → hidden gems → practical notes—each chapter **eyebrow + title + body**.
- **Partial guide:** **single contained statement** of incompleteness + **one** confident next step (deepest available guide), not an empty template.

**Why:** Empty chrome destroys editorial brands faster than short honest copy.

---

## 5. Trust / services / contact (utility-forward)

**Job:** Reduce anxiety and friction; **not** to win emotional peak moments.

**Pacing:** Clear headline → credibility markers → **scannable** actions → form or channels.

**Composition note:** Still use **section spacing discipline** and banded blocks for policy-ish content so it doesn’t become a wall.

---

## Anti-patterns by archetype

| Archetype | Failure mode |
|-----------|----------------|
| Magazine home | Hero overloaded with links and stats |
| Discovery list | Filters first, no “why these items exist” |
| Long-form detail | Specs and price before story |
| Dossier | Identical layout when content is missing |
| Utility | Ornate heroes that delay the task |
